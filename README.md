# Claude Code & Codex Practice

A practical playbook for using Claude Code and Codex effectively in real coding work.

## Goal

This repository is for fast onboarding and repeated reference when using AI coding agents in day-to-day engineering work.

It focuses on:

- how to choose between Claude Code and Codex
- high-value workflows that actually save time
- prompt patterns that are reusable
- review / debugging / refactor practices
- common pitfalls and guardrails

## Repository Structure

- `docs/01-overview.md` — what each tool is good at
- `docs/02-claude-code-practices.md` — Claude Code workflows and usage patterns
- `docs/03-codex-practices.md` — Codex workflows and usage patterns
- `docs/04-shared-playbook.md` — shared best practices for both tools
- `docs/05-daily-workflows.md` — workflows for feature work, debugging, review, and research
- `prompts/` — reusable prompt templates

## Suggested Way to Use This Repo

1. Read the overview first.
2. Pick one tool and try 2-3 workflows from the docs.
3. Save successful prompts into `prompts/`.
4. Keep updating the playbook based on real project experience.

## Practical Principles

- use AI for acceleration, not blind delegation
- constrain the task clearly
- ask for plans before large edits
- verify outputs with tests, diffs, and reviews
- treat generated code as draft code until verified
